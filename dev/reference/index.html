ConformalPrediction.ConformalModels.score — Method
score(conf_model::JackknifeRegressor, Xtrain, ytrain)

For the JackknifeRegressor prediction intervals are computed as follows,

$\begin{aligned} \hat{C}_{n,\alpha}(X_{n+1}) &= \hat\mu(X_{n+1}) \pm \hat{q}_{n, \alpha}^{+} \{|Y_i - \hat\mu_{-i}(X_i)|\}, \ i \in \mathcal{D}_{\text{train}} \end{aligned}$

where $\hat\mu_{-i}$ denotes the model fitted on training data with $i$th point removed. The jackknife procedure addresses the overfitting issue associated with the NaiveRegressor.

conf_model = conformal_model(model; method=:jackknife)
score(conf_model, X, y)
source
ConformalPrediction.ConformalModels.score — Method
score(conf_model::NaiveClassifier, Xtrain, ytrain)

For the NaiveClassifier prediction sets are computed as follows:

$\begin{aligned} \hat{C}_{n,\alpha}(X_{n+1}) &= \left\{y: s(X_{n+1},y) \le \hat{q}_{n, \alpha}^{+} \{|Y_i - \hat\mu(X_i) |\} \right\}, \ i \in \mathcal{D}_{\text{train}} \end{aligned}$

The naive approach typically produces prediction regions that undercover due to overfitting.

Examples

conf_model = conformal_model(model; method=:naive)
score(conf_model, X, y)

ConformalPrediction.ConformalModels.score — Method
score(conf_model::SimpleInductiveClassifier, Xtrain, ytrain)

For the SimpleInductiveClassifier prediction sets are computed as follows,

$\begin{aligned} \hat{C}_{n,\alpha}(X_{n+1}) &= \left\{y: s(X_{n+1},y) \le \hat{q}_{n, \alpha}^{+} \{|Y_i - \hat\mu(X_i) |\} \right\}, \ i \in \mathcal{D}_{\text{calibration}} \end{aligned}$

where $\mathcal{D}_{\text{calibration}}$ denotes the designated calibration data and $\hat\mu$ denotes the model fitted on training data $\mathcal{D}_{\text{train}}$.

Examples

conf_model = conformal_model(model; method=:simple)
score(conf_model, X, y)

MLJModelInterface.fit — Method
fit(conf_model::InductiveConformalModel, verbosity, X, y)

Wrapper function to fit the underlying MLJ model. For Inductive Conformal Prediction the underlying model is fitted on the proper training set. The fitresult is assigned to the model instance. Computation of nonconformity scores requires a separate calibration step involving a calibration data set (see calibrate!).
